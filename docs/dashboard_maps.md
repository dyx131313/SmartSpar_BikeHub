# dashboard maps logic
## 数据展示面板地图显示逻辑

### 1. 数据来源逻辑 (后端)

后端接口 `/api/dashboard/stations` 负责聚合所有需要的数据。它为每一个站点打包了三个关键指标：

*   **`current_bikes` (现存单车量)**：
    *   **逻辑**：查询 `BikeHistory` 表，获取该站点**最新一条**上报记录中的 `available_bikes`。
    *   **意义**：代表当前时刻，该站点停了多少辆车。
*   **`predicted_demand` (预测需求量)**：
    *   **逻辑**：查询 `Prediction` 表，获取该站点**最新一条** AI 预测记录中的 `predicted_demand`。
    *   **意义**：代表 AI 算法认为该站点在未来一段时间会有多少用车需求。
*   **`real_demand` (真实需求量)**：
    *   **逻辑**：查询 `DemandData` 表，获取该站点**最新一条**人工录入或系统采集的真实需求记录中的 `demand`。
    *   **意义**：代表实际发生的、或者人工确认的用车需求。

### 2. 地图展示逻辑 (前端)

前端地图组件 (`AmapComponent`) 根据您在 Tab 栏选择的模式 (`mode`)，决定如何渲染这些数据。目前共有四种模式：

#### A. 综合视图 (`combined`)
这是默认视图，旨在提供最全面的信息。
*   **热力图层**：显示 **`predicted_demand` (预测需求)**。
    *   *逻辑*：使用 AI 预测值作为权重。
    *   *目的*：让调度员一眼看到未来哪里最缺车（红色区域）。
*   **图钉层 (Markers)**：显示 **`current_bikes` (现存单车)**。
    *   *逻辑*：在地图上打点，点击或查看图钉可以看到具体的车辆数。
    *   *目的*：让调度员看到哪里有车可以调。

#### B. 预测需求热力图 (`heatmap`)
*   **显示内容**：仅显示热力图。
*   **数据源**：**`predicted_demand` (预测需求)**。
*   **用途**：专注于查看 AI 的预测结果，不受站点图钉遮挡，分析未来的供需趋势。

#### C. 真实需求热力图 (`real_demand_heatmap`)
*   **显示内容**：仅显示热力图。
*   **数据源**：**`real_demand` (真实需求)**。
*   **用途**：
    *   查看历史真实发生的“热区”。
    *   **验证刚才的操作**：当您在“需求数据管理”页面手动录入一条需求后，切换到这个视图，就能立即看到您录入的数据反映在热力图上（因为这是直接读 `DemandData` 表）。

#### D. 图钉图 (`markers`)
*   **显示内容**：仅显示站点图钉。
*   **数据源**：**`current_bikes` (现存单车)**。
*   **用途**：
    *   专注于车辆分布。
    *   当热力图颜色过深影响查看具体站点位置时，使用此模式可以清晰地看到每个站点的确切位置和车辆数。

### 总结对照表

| 视图模式 | 热力图数据源 | 图钉显示内容 | 典型使用场景 |
| :--- | :--- | :--- | :--- |
| **综合视图** | **预测需求** (AI) | 现存单车 | 宏观调度，同时看“哪里缺车”和“哪里有车” |
| **预测热力图** | **预测需求** (AI) | 无 | 分析 AI 预测趋势，制定未来调度计划 |
| **真实热力图** | **真实需求** (人工/历史) | 无 | 复盘历史数据，或查看人工录入的突发需求 |
| **图钉图** | 无 | 现存单车 | 查找具体站点，查看精确车辆库存 |
