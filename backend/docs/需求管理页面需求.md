# 需求数据运营后台 – 前端需求规格说明书

> 对应后端：`backend/routers/demand_data.py`（7 个接口）  
> 文档版本：v1.0  
> 适用角色：admin / dispatcher / user / operator  

---

## 1 总体目标

构建 **“需求数据运营后台”** 最小可用版本，覆盖需求数据的 **增删改查、批量导入、统计可视化** 三大场景，保持与后端接口 1:1 映射。

---

## 2 页面总览

| 一级菜单 | 二级页面 | 路由（建议） | 核心接口 |
| --- | --- | --- | --- |
| 需求数据管理 | 需求数据列表 | `/demand/list` | `GET /demand-data` |
|  | 批量导入 | `/demand/import` | `POST /demand-data/import` |
|  | 数据报表 | `/demand/statistics` | `GET /demand-data/statistics` |

（*仅列表页含弹窗，其余为独立页面*）

## 3 页面详情

### 3.1 需求数据列表页 `/demand/list`

#### 3.1.1 查询区域（可折叠）
| 字段 | 组件 | 参数 key | 必填 | 备注 |
| --- | --- | --- | --- | --- |
| 时间范围 | RangePicker | startDate / endDate | × | 精确到分钟 |
| 站点类型 | Select | stationType | × | 静态字典 |
| 天气 | Select | weather | × | sunny/rainy/cloudy… |
| 星期 | Select | weekday | × | 0-6 |

> 点击 **查询** 触发 `GET /demand-data` 并带回分页参数。

#### 3.1.2 按钮组
| 按钮 | 可见角色 | 动作 |
| --- | --- | --- |
| 新增 | admin / dispatcher | 打开新增弹窗 |
| 批量删除 | admin / dispatcher | 二次确认后调用 `DELETE /demand-data?id=1,2,3` |
| 导出 JSON | 全部 | 下载当前筛选结果 `demand_YYYYMMDD_HHMMSS.json` |
| 刷新 | 全部 | 重新请求当前页 |

#### 3.1.3 表格（ProTable）
| 列 | 渲染 | 操作 |
| --- | --- | --- |
| 时间 | 格式 `YYYY-MM-DD HH:mm` | — |
| 站点 | 显示 name（下拉已做 id→name 映射） | — |
| 站点类型 | tag | — |
| 需求值 | 数字 | — |
| 天气 | tag | — |
| 操作 | 按钮组 | 查看 / 编辑 / 删除（二次确认） |

**分页参数**：`page & pageSize`（默认 20，最大 100）  
**记忆**：离开页面再返回保持 `page / pageSize / 查询条件`

#### 3.1.4 操作弹窗 / 抽屉
- **查看**：抽屉，仅读
- **编辑**：同新增弹窗，标题改为“编辑”
- **删除**：Popconfirm → 调用 `DELETE /demand-data/<id>`

---

### 3.2 新增 / 编辑弹窗（复用组件 `<DemandFormModal>`）

| 字段 | 组件 | 校验 | 备注 |
| --- | --- | --- | --- |
| 时间戳 | DateTimePicker | 必填 | 默认当前小时整点 |
| 站点 | Select（远程） | 必填 | `GET /stations` id→name |
| 站点类型 | Input | 只读 | 选站点后自动带出 |
| 星期 | Select | 可空 | 可自动按日期计算 |
| 节假日 | Checkbox | — | — |
| 天气 | Select | 必填 | 静态字典 |
| 温度 | InputNumber | 可空 | 单位 ℃ |
| 需求值 | InputNumber | 必填 >0 | 单位：车/人次/吨 视业务 |

**提交**  
新增 → `POST /demand-data`  
编辑 → `PUT /demand-data/<id>`  
成功：关闭弹窗 + 刷新列表 + toast 提示

---

### 3.3 批量导入页 `/demand/import`

| 区域 | 说明 |
| --- | --- |
| 上传卡片 | Dragger 上传，仅 `.json` 格式 |
| 模板下载 | 提供 `template_demand.json`（前端 public 静态文件） |
| 导入结果 | 表格：成功条数 / 跳过条数 / 前 10 条错误原因 |
| 重新上传 | 失败后一键清空重新选择 |

**调用**  
`POST /demand-data/import` form-data（key=`file`）  
上传过程加 **loading 遮罩**；后端返回：

```json
{
  "imported_count": 2990,
  "skipped_count": 10,
  "errors": ["数据项 2025-12-03T14:00:00 导入失败: station_id 99 不存在", ...]
}
```

### 3.4 数据报表页 `/demand/statistics`

#### 3.4.1 筛选区
与列表页共用 **时间范围 + 站点类型** 组件，参数同 `GET /demand-data/statistics`

#### 3.4.2 指标卡片
| 指标 | 字段 |
| --- | --- |
| 总记录数 | totalRecords |
| 平均需求 | averageDemand |
| 最大需求 | maxDemand |
| 最小需求 | minDemand |

#### 3.4.3 图表
| 图表 | 类型 | 数据 key |
| --- | --- | --- |
| 按站点类型 | 柱状图 | stationTypeStats[] |
| 按天气 | 饼图 | weatherStats[] |

**空状态**：无数据时显示灰色占位图 + “暂无数据”文案。

## 4 权限控制（按钮级）

| 角色 | 新增/编辑/删除/导入 | 导出 | 查看 |
| --- | --- | --- | --- |
| admin / dispatcher | ✔ | ✔ | ✔ |
| user / operator | ✘ | ✔ | ✔ |

实现方式：  
前端解析 JWT payload 中 `roles` 字段，动态控制按钮 `v-if` / `disabled`。

---

## 5 UX 细则

1. **分页记忆**：`page & pageSize` 存 sessionStorage，返回列表自动恢复。
2. **二次确认**：删除/批量删除使用 Popconfirm。
3. **上传体验**：大文件显示进度条（Ant Upload `onProgress`）。
4. **错误提示**：所有后端 400/500 统一封装为消息条（message.error）。
5. **导出文件名**：`demand_YYYYMMDD_HHmmss.json`，与筛选条件一致。

## 6 接口清单（前后映射）

| 前端场景 | 方法 + URL | 备注 |
| --- | --- | --- |
| 列表/查询 | `GET /demand-data` | 分页、过滤 |
| 单条新增 | `POST /demand-data` | payload = object |
| 单条编辑 | `PUT /demand-data/{id}` | 部分字段 |
| 单条删除 | `DELETE /demand-data/{id}` | 物理删除 |
| 批量导入 | `POST /demand-data/import` | form-data file |
| 统计数据 | `GET /demand-data/statistics` | 卡片 + 图表 |
| 站点下拉 | `GET /stations` | 新增/编辑弹窗远程搜索 |
