## 通用信息

### 请求格式
- **Content-Type**: `application/json`
- **认证方式**: Bearer Token (JWT)
- **字符编码**: UTF-8

### 响应格式
所有接口统一返回JSON格式：

#### 成功响应格式
```json
{
  "message": "操作成功",
  "data": {
    // 具体数据
  },
  "total": 100,        // 仅分页接口
  "pages": 5,          // 仅分页接口
  "current_page": 1,   // 仅分页接口
  "per_page": 20       // 仅分页接口
}
```

#### 错误响应格式
```json
{
  "error": "错误描述信息"
}
```

### 认证说明
除登录接口外，所有接口都需要在请求头中携带JWT Token：

```
Authorization: Bearer <your_jwt_token>
```

## A*算法路径规划模块

### 概述

A*算法路径规划模块为智慧共享单车调度系统提供智能路径规划功能，支持单点路径规划和多点路径规划，能够为运维车辆提供最优/次优路径方案。

### 功能特性

- **A*算法实现**: 基于Haversine距离的启发式搜索
- **单点路径规划**: 计算两个站点间的最优路径
- **多点路径规划**: 支持途径点的TSP问题近似解
- **距离计算**: 精确的地理坐标距离计算
- **路径优化**: 支持最短距离、最短时间等多种优化目标
- **历史记录**: 路径执行历史和反馈收集

### 技术特点

- **高效算法**: A*算法保证找到最优路径
- **地理计算**: 基于Haversine公式的精确距离计算
- **灵活配置**: 支持自定义连接距离和优化目标
- **完整记录**: 路径规划、执行、反馈的完整生命周期管理

### 1. 创建路径规划

**接口地址**: `POST /api/route-planning/plan`

**请求参数**:

## 算法原理

### A*算法
A*算法是一种启发式搜索算法，结合了Dijkstra算法的准确性和贪心最佳优先搜索的效率。

**核心公式**: `f(n) = g(n) + h(n)`
- `g(n)`: 从起点到节点n的实际代价
- `h(n)`: 从节点n到终点的启发式代价（估计值）
- `f(n)`: 节点n的总估计代价

### 启发式函数
使用Haversine公式计算两点间的球面距离作为启发式函数，保证精确的地理距离计算。

### 多点路径规划
采用贪心算法解决TSP问题的近似解：
1. 从起点开始
2. 每次选择距离当前位置最近的未访问点
3. 重复直到所有点都被访问
4. 最后返回终点

## 数据类型说明

### 路径状态 (status)
- `planned`: 已规划
- `in_progress`: 进行中
- `completed`: 已完成
- `cancelled`: 已取消

### 优化目标 (optimization_goal)
- `shortest`: 最短距离
- `fastest`: 最短时间
- `balanced`: 平衡优化

### 完成状态 (completion_status)
- `success`: 成功完成
- `partial`: 部分完成
- `failed`: 执行失败

### 交通状况 (traffic_conditions)
- `light`: 畅通
- `moderate`: 适中
- `heavy`: 拥堵

## 性能指标

### 算法复杂度
- **时间复杂度**: O(b^d)，其中b是分支因子，d是解的深度
- **空间复杂度**: O(b^d)

### 系统要求
- **支持站点数**: 无限制
- **路径规划时间**: < 100ms（10个站点内）
- **并发支持**: 支持50个用户同时使用
- **内存占用**: < 100MB
- **途径点数**: ≥10个（满足系统要求）
